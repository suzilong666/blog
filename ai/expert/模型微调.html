模型微调基础 ​

什么是模型微调？ ​

模型微调（Fine-tuning）是指在预训练大模型的基础上，使用特定领域的数据进行进一步训练，使模型更好地适应特定任务或领域。

为什么需要微调？ ​

预训练模型的局限性：

  • 通用性强但专业性不足
  • 缺乏特定领域的知识
  • 风格不符合企业需求
  • 某些任务效果不佳

微调的优势：

  • ✅ 提升特定任务性能
  • ✅ 注入领域知识
  • ✅ 定制化输出风格
  • ✅ 减少 Prompt 长度
  • ✅ 降低推理成本

微调 vs Prompt Engineering ​

特性Prompt Engineering微调
实现难度低中到高
数据需求无需要标注数据
成本低中高
效果提升有限显著
适用场景快速验证生产环境

微调方法概述 ​

1. 全量微调（Full Fine-tuning） ​

更新模型的所有参数。

优点：

  • 效果最好
  • 完全适配目标任务

缺点：

  • 计算资源需求巨大
  • 容易过拟合
  • 存储成本高
python
# 伪代码示例
model = AutoModelForCausalLM.from_pretrained("llama-3-8b")

# 解冻所有参数
for param in model.parameters():
    param.requires_grad = True

# 训练
trainer = Trainer(
    model=model,
    train_dataset=train_data,
    args=TrainingArguments(...)
)
trainer.train()

2. LoRA（Low-Rank Adaptation） ​

只训练低秩分解矩阵，冻结原始模型参数。

优点：

  • 参数量少（约 1%）
  • 训练速度快
  • 易于切换不同任务
  • 存储成本低

缺点：

  • 效果略低于全量微调
python
from peft import LoraConfig, get_peft_model

# 配置 LoRA
lora_config = LoraConfig(
    r=16,  # 秩
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],  # 目标层
    lora_dropout=0.05,
    bias="none"
)

# 应用 LoRA
model = AutoModelForCausalLM.from_pretrained("llama-3-8b")
model = get_peft_model(model, lora_config)

# 训练（只训练 LoRA 参数）
trainer = Trainer(model=model, ...)

3. QLoRA（Quantized LoRA） ​

结合量化和 LoRA，进一步降低资源需求。

特点：

  • 4-bit 量化 + LoRA
  • 可在消费级 GPU 上训练
  • 内存占用极低
python
from bitsandbytes import BitsAndBytesConfig

# 4-bit 量化配置
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.float16
)

# 加载量化模型
model = AutoModelForCausalLM.from_pretrained(
    "llama-3-8b",
    quantization_config=bnb_config
)

# 应用 LoRA
model = get_peft_model(model, lora_config)

何时需要微调？ ​

适合微调的场景 ​

✅ 需要微调：

  1. 专业领域问答（医疗、法律、金融）
  2. 特定风格的文本生成
  3. 结构化输出（JSON、SQL）
  4. 企业内部术语和流程
  5. 高频使用的固定任务

❌ 不需要微调：

  1. 通用问答
  2. 临时性任务
  3. 数据量太少（<100 条）
  4. Prompt Engineering 已能满足需求

数据收集 ​

python
import json

# 从对话日志构建数据集
def build_dataset(conversations: list) -> list:
    """将对话转换为微调格式"""
    dataset = []
    
    for conv in conversations:
        messages = [
            {"role": "system", "content": "你是专业的客服助手"}
        ]
        
        for turn in conv["turns"]:
            messages.append({
                "role": "user" if turn["sender"] == "user" else "assistant",
                "content": turn["message"]
            })
        
        dataset.append({"messages": messages})
    
    return dataset

# 保存为 JSONL
def save_jsonl(data: list, filename: str):
    with open(filename, 'w', encoding='utf-8') as f:
        for item in data:
            f.write(json.dumps(item, ensure_ascii=False) + '\n')

# 使用
dataset = build_dataset(conversations)
save_jsonl(dataset, "training_data.jsonl")

数据质量要求 ​

最少数据量：

  • GPT-3.5-turbo：至少 50-100 条
  • 建议：500+ 条以获得好效果

质量标准：

  1. 答案准确无误
  2. 风格一致
  3. 覆盖常见场景
  4. 包含边界案例

Hugging Face 微调实战 ​

使用 Transformers ​

python
from transformers import (
    AutoTokenizer,
    AutoModelForCausalLM,
    TrainingArguments,
    Trainer
)
from datasets import load_dataset
from peft import LoraConfig, get_peft_model

# 1. 加载数据和模型
dataset = load_dataset("json", data_files="training_data.jsonl")
model_name = "meta-llama/Llama-3-8b"

tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(model_name)

# 2. 配置 LoRA
lora_config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05
)

model = get_peft_model(model, lora_config)

# 3. 数据预处理
def tokenize_function(examples):
    return tokenizer(
        examples["text"],
        truncation=True,
        max_length=512,
        padding="max_length"
    )

tokenized_dataset = dataset.map(tokenize_function, batched=True)

# 4. 训练配置
training_args = TrainingArguments(
    output_dir="./results",
    num_train_epochs=3,
    per_device_train_batch_size=4,
    gradient_accumulation_steps=4,
    learning_rate=2e-4,
    fp16=True,
    logging_steps=10,
    save_strategy="epoch"
)

# 5. 开始训练
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=tokenized_dataset["train"]
)

trainer.train()

# 6. 保存模型
model.save_pretrained("./fine_tuned_model")
tokenizer.save_pretrained("./fine_tuned_model")

def evaluate_model(model, test_questions: list) -> dict:
    """人工评估模型表现"""
    
    results = []

        response = generate_response(model, question)
        
        # 人工评分（1-5分）

        print(f"回答: {response}")
        score = int(input("评分 (1-5): "))
        
        results.append({
            "question": question,
            "response": response,
            "score": score
        })
    
    avg_score = sum(r["score"] for r in results) / len(results)
    
    return {
        "average_score": avg_score,
        "results": results
    }

2. 自动指标 ​

python
from rouge_score import rouge_scorer

def calculate_rouge(predictions: list, references: list) -> dict:
    """计算 ROUGE 分数"""
    
    scorer = rouge_scorer.RougeScorer(['rouge1', 'rouge2', 'rougeL'], use_stemmer=True)
    
    scores = {
        "rouge1": [],
        "rouge2": [],
        "rougeL": []
    }
    
    for pred, ref in zip(predictions, references):
        score = scorer.score(ref, pred)
        for key in scores:
            scores[key].append(score[key].fmeasure)
    
    # 平均分
    return {key: sum(val)/len(val) for key, val in scores.items()}

python
# OpenAI 微调成本估算
def estimate_finetune_cost(num_tokens: int, n_epochs: int = 3) -> float:
    """估算微调成本"""
    # GPT-3.5-turbo: $0.008 / 1K tokens
    cost_per_1k = 0.008
    
    total_tokens = num_tokens * n_epochs
    cost = (total_tokens / 1000) * cost_per_1k
    
    return round(cost, 2)

# 示例：10万 tokens，3个 epoch
cost = estimate_finetune_cost(100000, 3)
print(f"预计成本: ${cost}")  # $2.40

常见问题 ​

Q1: 需要多少数据？ ​

  • 最少：50-100 条高质量样本
  • 推荐：500-1000 条
  • 理想：2000+ 条

Q2: 微调后效果不好怎么办？ ​

  1. 检查数据质量
  2. 增加数据量
  3. 调整超参数
  4. 尝试不同的基础模型

Q3: 如何管理多个微调模型？ ​

python
# 为不同任务使用不同的模型
models = {
    "customer_service": "ft:gpt-3.5-turbo:cs-v1",
    "code_assistant": "ft:gpt-3.5-turbo:code-v1",
    "data_analysis": "ft:gpt-3.5-turbo:data-v1"
}

def get_model(task: str):
    return models.get(task, "gpt-3.5-turbo")

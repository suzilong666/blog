Prompt 工程基础 ​

什么是 Prompt ​

Prompt（提示词）是你输入给大模型的文本指令，用于引导模型生成期望的输出。可以把 Prompt 理解为与 AI 对话的"问题"或"指令"，它的质量直接决定了回答的质量。

核心理念：Garbage In, Garbage Out（垃圾进，垃圾出）

糟糕的 Prompt 产生糟糕的结果，优秀的 Prompt 产生优秀的结果。

Prompt 的基本结构 ​

一个完整的 Prompt 通常包含以下要素：

  1. 角色设定 (Role) - 为模型指定身份
  2. 任务描述 (Task) - 清晰说明任务
  3. 上下文背景 (Context) - 提供必要信息
  4. 约束条件 (Constraints) - 明确限制
  5. 输出格式 (Format) - 指定期望形式
  6. 示例参考 (Examples) - 提供参考样例（可选）

示例对比 ​

糟糕的 Prompt：

写点关于人工智能的东西

优秀的 Prompt：

你是一位科普作家。请用通俗易懂的语言，向没有技术背景的普通人解释什么是人工智能。

要求：
- 使用生活中的类比
- 避免专业术语
- 控制在 300 字以内
- 包含一个具体的应用场景

常用 Prompt 技巧 ​

1. Zero-Shot Prompting（零样本提示） ​

不提供示例，直接让模型完成任务。

将以下英文翻译成中文：
"Artificial intelligence is transforming the world."

2. Few-Shot Prompting（少样本提示） ​

提供少量示例，帮助模型理解任务模式。

根据以下示例，将情感分类为"正面"、"负面"或"中性"：

示例1：
输入："这部电影太棒了！"
输出：正面

示例2：
输入："服务很差，不会再来了。"
输出：负面

现在请分类：
输入："产品质量一般，价格偏贵。"
输出：

3. Chain-of-Thought（思维链） ​

引导模型逐步推理，特别适用于复杂问题。

请逐步思考并解答：

问题：小明有5个苹果，他吃了2个，又买了3个，现在有多少个？

解答步骤：
1. 初始数量：5个
2. 吃了2个：5 - 2 = 3个
3. 又买了3个：3 + 3 = 6个
4. 最终答案：6个

请用同样的方式解答：
问题：一个班级有30名学生，其中60%是女生，男生有多少人？

4. Role Prompting（角色扮演） ​

赋予模型特定角色，获得更专业的回答。

你是一位资深软件架构师，正在审查初级开发者的代码。
请指出以下代码的问题，并给出改进建议：

[代码内容]

最佳实践 ​

应该做的 ​

  1. 具体明确 - 避免模糊不清的指令
  2. 结构化表达 - 使用列表、分段组织
  3. 提供上下文 - 帮助模型理解背景
  4. 指定期望格式 - 明确输出形式
  5. 迭代优化 - 根据结果调整 Prompt

不应该做的 ​

  1. 避免模糊不清 - "给我一些好的想法"
  2. 避免过度复杂 - 不要一次性要求太多
  3. 避免矛盾指令 - "详细解释但要非常简短"
  4. 不要假设模型知道 - 重新描述完整要求

常见错误与修复 ​

错误1：指令不够具体 ​

修复前：

帮我写代码

修复后：

请用Python编写一个函数，接收一个列表作为参数，返回去重后的列表，保持原有顺序。
要求：
- 时间复杂度 O(n)
- 添加类型注解
- 包含docstring和使用示例

错误2：忽略上下文 ​

修复前：

这个怎么改？

修复后：

我在开发一个用户登录系统，遇到以下问题：
- 技术栈：React + Node.js + MongoDB
- 问题：密码验证总是失败

以下是相关代码：
[代码内容]

请帮我找出问题所在。

错误3：没有指定输出格式 ​

修复前：

比较一下React和Vue

修复后：

请对比 React 和 Vue 框架，以表格形式呈现，包含以下维度：
- 学习曲线
- 社区规模
- 性能表现
- 适用场景

实用模板 ​

代码生成模板 ​

角色：你是一位资深的[编程语言]开发工程师
任务：编写一个函数，实现[功能描述]
要求：
- 输入：[输入描述]
- 输出：[输出描述]
- 边界情况：[边界情况]

请提供完整代码、注释和使用示例。

代码审查模板 ​

角色：你是一位严格的代码审查员
待审查代码：[代码内容]

请从以下角度审查：
1. 正确性：是否有bug或逻辑错误
2. 性能：是否存在性能瓶颈
3. 可读性：代码是否清晰易懂
4. 安全性：是否有安全隐患

输出：问题列表、改进建议、优化后的代码、评分(1-10)

学习解释模板 ​

角色：你是一位善于教学的[学科]老师
学生背景：[学生背景描述]
要解释的概念：[概念名称]

教学要求：
- 使用生活化的类比
- 避免专业术语
- 由浅入深，循序渐进
- 包含具体例子

高级技巧 ​

1. 分隔符的使用 ​

使用特殊符号清晰分隔不同部分：

请分析以下代码的问题：

=== 代码开始 ===
def calculate_sum(numbers):
    total = 0
    for num in numbers:
        total += num
    return total
=== 代码结束 ===

请从代码风格、性能优化、边界处理三个角度分析。

2. 自我验证 ​

要求模型检查自己的回答：

请解答这个问题，然后：
1. 验证你的答案是否正确
2. 检查是否有遗漏的情况
3. 如果有错误，请修正
4. 给出最终确定的答案

3. 负向提示 ​

明确告诉模型不要做什么：

请总结这篇文章的主要观点。

要求：
- 简洁明了，不超过100字
- 不要包含个人观点
- 不要引用原文
- 不要添加额外信息

小结 ​

Prompt Engineering 是与大模型高效沟通的核心技能。

核心原则：

  • 具体明确，避免模糊
  • 结构清晰，层次分明
  • 提供上下文和示例
  • 迭代优化，持续改进

常用技巧：

  • Zero-shot / Few-shot prompting
  • Chain-of-thought reasoning
  • Role prompting
  • Template-based prompting

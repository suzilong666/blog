# 内存管理

[官方文档](https://docs.python.org/zh-cn/3/c-api/refcounting.html)

Python 使用自动内存管理机制，主要包括引用计数和垃圾回收。理解内存管理有助于编写更高效的代码，避免内存泄漏。本文将介绍 Python 的内存管理机制。

## 引用计数

### 基本原理

每个对象都有一个引用计数器，记录有多少引用指向它。

```python
import sys

a = []
print(sys.getrefcount(a))  # 2（a + getrefcount 参数）

b = a
print(sys.getrefcount(a))  # 3

del b
print(sys.getrefcount(a))  # 2

del a
# 引用计数为 0，对象被销毁
```

### 循环引用

```python
# 循环引用导致引用计数无法清理
class Node:
    def __init__(self):
        self.ref = None

a = Node()
b = Node()
a.ref = b
b.ref = a

del a
del b
# 引用计数不为 0，需要垃圾回收处理
```

## 垃圾回收

### gc 模块

```python
import gc

# 手动触发垃圾回收
gc.collect()

# 查看垃圾回收统计
print(gc.get_stats())

# 禁用/启用垃圾回收
gc.disable()
gc.enable()
```

### 分代回收

Python 将对象分为三代：
- **第 0 代**：新创建的对象
- **第 1 代**：存活一次回收的对象
- **第 2 代**：存活多次回收的对象

```python
import gc

# 查看阈值
print(gc.get_threshold())  # (700, 10, 10)

# 设置阈值
gc.set_threshold(700, 10, 10)
```

## 内存泄漏检测

### tracemalloc 模块

```python
import tracemalloc

# 开始跟踪
tracemalloc.start()

# 你的代码
data = [i for i in range(100000)]

# 获取快照
snapshot = tracemalloc.take_snapshot()
top_stats = snapshot.statistics('lineno')

print("[ Top 10 ]")
for stat in top_stats[:10]:
    print(stat)

# 停止跟踪
tracemalloc.stop()
```

## 优化建议

### 1. 使用生成器

```python
# 不好：一次性加载所有数据
data = [x ** 2 for x in range(1000000)]

# 好：惰性计算
data = (x ** 2 for x in range(1000000))
```

### 2. 及时释放资源

```python
# 使用 with 语句
with open("large_file.txt") as f:
    content = f.read()
# 文件自动关闭，释放资源
```

### 3. 避免全局变量

```python
# 不好
global_data = []

def process():
    global global_data
    global_data.append(large_object)

# 好：局部变量
def process():
    local_data = []
    local_data.append(large_object)
    return local_data
```

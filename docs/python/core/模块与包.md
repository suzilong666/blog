# 模块与包

[官方文档](https://docs.python.org/zh-cn/3/tutorial/modules.html)

随着代码量增加，我们需要将代码组织到多个文件中。Python 使用模块和包来管理代码，提供命名空间隔离和代码复用机制。本文将介绍 Python 的模块系统和包的创建方法。

## 模块基础

### 什么是模块？

模块是一个包含 Python 代码的 `.py` 文件，可以定义函数、类和变量。

```python
# math_utils.py
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

PI = 3.14159
```

### 导入模块

```python
# main.py

# 方式1：导入整个模块
import math_utils
result = math_utils.add(5, 3)
print(result)  # 8

# 方式2：导入特定函数
from math_utils import add, subtract
result = add(5, 3)
print(result)  # 8

# 方式3：导入所有内容（不推荐）
from math_utils import *
result = add(5, 3)

# 方式4：使用别名
import math_utils as mu
result = mu.add(5, 3)
```

> 💡 **最佳实践**：优先使用 `import module` 或 `from module import specific_item`，避免 `import *`。

## 模块搜索路径

Python 按照以下顺序查找模块：

1. 当前目录
2. `PYTHONPATH` 环境变量中的目录
3. 安装目录（site-packages）

```python
import sys

# 查看模块搜索路径
for path in sys.path:
    print(path)

# 临时添加路径
sys.path.append("/my/custom/path")
```

## 模块的属性

每个模块都有特殊属性：

```python
# my_module.py
print(__name__)  # 模块名

if __name__ == "__main__":
    # 直接运行时执行
    print("作为主程序运行")
else:
    # 被导入时执行
    print("被导入为模块")
```

### 实际应用

```python
# calculator.py
def add(a, b):
    return a + b

def test():
    assert add(2, 3) == 5
    print("测试通过")

if __name__ == "__main__":
    test()  # 只在直接运行时执行测试
```

## 包（Package）

包是包含多个模块的目录，通过 `__init__.py` 文件标识。

### 创建包

```
my_package/
├── __init__.py
├── module_a.py
├── module_b.py
└── subpackage/
    ├── __init__.py
    └── module_c.py
```

### 导入包中的模块

```python
# 导入包中的模块
from my_package import module_a
from my_package.subpackage import module_c

# 导入特定函数
from my_package.module_a import some_function

# 使用点号访问
import my_package.module_a
my_package.module_a.some_function()
```

## __init__.py 的作用

`__init__.py` 可以是空文件，也可以包含初始化代码。

```python
# my_package/__init__.py

# 方式1：空文件（仅标识为包）

# 方式2：导出常用接口
from .module_a import add, subtract
from .module_b import multiply

# 方式3：包级别的初始化
print("包已加载")

# 方式4：定义 __all__
__all__ = ["add", "subtract", "multiply"]
```

### 简化导入

```python
# 用户可以直接从包导入
from my_package import add, subtract, multiply

# 而不需要
# from my_package.module_a import add, subtract
# from my_package.module_b import multiply
```

## 相对导入与绝对导入

### 绝对导入

从包的根目录开始导入。

```python
# my_package/subpackage/module_c.py
from my_package.module_a import add  # 绝对导入
```

### 相对导入

使用 `.` 表示当前包，`..` 表示父包。

```python
# my_package/subpackage/module_c.py
from ..module_a import add  # 相对导入（上一级）
from . import helper        # 相对导入（当前包）
```

> 💡 **最佳实践**：优先使用绝对导入，更清晰易读。

## 常见错误

### 1. 循环导入

```python
# module_a.py
from module_b import func_b  # 错误！

def func_a():
    pass

# module_b.py
from module_a import func_a  # 错误！

def func_b():
    pass
```

**解决**：重构代码，消除循环依赖。

### 2. 模块未找到

```python
# ImportError: No module named 'my_module'

# 检查：
# 1. 文件是否存在
# 2. 是否在搜索路径中
# 3. 是否有拼写错误
```

### 3. 名称冲突

```python
# 不要用自己的文件名覆盖标准库
# json.py - 会覆盖标准库 json 模块！
# email.py - 会覆盖标准库 email 模块！
```

## 实际应用示例

### 1. 项目结构

```
my_project/
├── src/
│   ├── __init__.py
│   ├── models.py
│   ├── views.py
│   └── controllers.py
├── tests/
│   ├── __init__.py
│   └── test_models.py
├── config.py
└── main.py
```

### 2. 配置模块

```python
# config.py
DATABASE_URL = "sqlite:///db.sqlite3"
DEBUG = True
SECRET_KEY = "your-secret-key"

# main.py
import config

if config.DEBUG:
    print("调试模式")
```

### 3. 工具包

```python
# utils/__init__.py
from .string_utils import capitalize, lowercase
from .math_utils import add, subtract

__all__ = ["capitalize", "lowercase", "add", "subtract"]

# utils/string_utils.py
def capitalize(text):
    return text.capitalize()

def lowercase(text):
    return text.lower()

# 使用
from utils import capitalize, add
```

## 最佳实践

1. **模块化设计**：每个模块职责单一
2. **清晰的命名**：模块名用小写+下划线
3. **避免循环导入**：合理设计依赖关系
4. **使用 __all__**：明确导出接口
5. **文档字符串**：为模块添加说明

```python
"""
数学工具模块

提供基本的数学运算功能。
"""

__all__ = ["add", "subtract"]

def add(a, b):
    """加法"""
    return a + b
```

# 代码规范

[PEP 8](https://peps.python.org/pep-0008/)

遵循代码规范可以提高代码可读性和可维护性。Python 的官方规范是 PEP 8。本文将介绍 PEP 8 的核心规则和常用工具。

## PEP 8 核心规则

### 命名规范

```python
# 变量和函数：小写+下划线
my_variable = 10
def my_function():
    pass

# 类：大驼峰
class MyClass:
    pass

# 常量：大写
MAX_SIZE = 100

# 私有：单下划线前缀
_private = "internal"
```

### 缩进

```python
# 使用 4 个空格
def function():
    if True:
        print("indented")

# 不要混用 tab 和空格
```

### 行长度

```python
# 每行不超过 79 字符
# 长表达式可以换行
result = (
    some_long_function_name(arg1, arg2)
    + another_function(arg3, arg4)
)
```

### 空行

```python
# 顶级函数/类之间：2 行
class ClassA:
    pass


class ClassB:
    pass


# 方法之间：1 行
class MyClass:
    def method1(self):
        pass

    def method2(self):
        pass
```

### 导入

```python
# 分组导入：标准库、第三方、本地
import os
import sys

import requests
import numpy as np

from my_module import my_function
```

## 工具

### black - 自动格式化

```bash
pip install black
black my_script.py
```

### flake8 - 代码检查

```bash
pip install flake8
flake8 my_script.py
```

### isort - 导入排序

```bash
pip install isort
isort my_script.py
```

## 最佳实践

1. **使用工具**：black、flake8、isort
2. **保持一致**：团队统一规范
3. **编写文档**：docstring 说明功能
4. **注释清晰**：解释为什么，而不是做什么
5. **定期审查**：code review 发现规范问题

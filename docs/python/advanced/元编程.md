# 元编程

[官方文档](https://docs.python.org/zh-cn/3/reference/datamodel.html#metaclasses)

元编程是指创建和操作类的代码。Python 中，类也是对象，可以被动态创建和修改。元类（Metaclass）是"类的类"，控制类的创建过程。本文将介绍 type()、元类和动态创建类的方法。

## type() 函数

### 查看类型

```python
# type() 可以查看对象的类型
print(type(42))        # <class 'int'>
print(type("hello"))   # <class 'str'>
print(type([1, 2]))    # <class 'list'>

# 类本身也是 type 的实例
class MyClass:
    pass

print(type(MyClass))   # <class 'type'>
```

### 动态创建类

```python
# type(name, bases, dict) 动态创建类
MyClass = type('MyClass', (), {
    'x': 10,
    'greet': lambda self: f"Hello, x={self.x}"
})

obj = MyClass()
print(obj.x)          # 10
print(obj.greet())    # Hello, x=10
```

### 带继承的动态类

```python
class Base:
    def base_method(self):
        return "Base method"

Child = type('Child', (Base,), {
    'child_method': lambda self: "Child method"
})

obj = Child()
print(obj.base_method())   # Base method
print(obj.child_method())  # Child method
```

## 元类概念

### 什么是元类？

元类是创建类的类。默认元类是 `type`。

```python
class Meta(type):
    def __new__(cls, name, bases, attrs):
        print(f"创建类: {name}")
        return super().__new__(cls, name, bases, attrs)

class MyClass(metaclass=Meta):
    pass
# 输出: 创建类: MyClass
```

### 元类的作用

1. **验证类定义**：确保类符合规范
2. **自动注册**：自动将类添加到注册表
3. **修改类属性**：添加或修改方法
4. **单例模式**：控制实例创建

## __new__ 方法

`__new__` 在 `__init__` 之前调用，负责创建对象。

```python
class Singleton(type):
    _instances = {}

    def __call__(cls, *args, **kwargs):
        if cls not in cls._instances:
            cls._instances[cls] = super().__call__(*args, **kwargs)
        return cls._instances[cls]

class Database(metaclass=Singleton):
    def __init__(self):
        print("初始化数据库连接")

db1 = Database()  # 初始化数据库连接
db2 = Database()  # 不会再次初始化
print(db1 is db2)  # True
```

## 实际应用示例

### 1. 自动注册插件

```python
class PluginRegistry(type):
    plugins = {}

    def __new__(cls, name, bases, attrs):
        new_class = super().__new__(cls, name, bases, attrs)
        if name != 'BasePlugin':
            PluginRegistry.plugins[name] = new_class
        return new_class

class BasePlugin(metaclass=PluginRegistry):
    pass

class AuthPlugin(BasePlugin):
    def authenticate(self):
        return "Authenticating..."

class LogPlugin(BasePlugin):
    def log(self, message):
        return f"Logging: {message}"

# 查看所有注册的插件
print(PluginRegistry.plugins)
# {'AuthPlugin': <class 'AuthPlugin'>, 'LogPlugin': <class 'LogPlugin'>}
```

### 2. ORM 框架

```python
class ModelMeta(type):
    def __new__(cls, name, bases, attrs):
        # 自动添加表名
        if 'table_name' not in attrs:
            attrs['table_name'] = name.lower()
        return super().__new__(cls, name, bases, attrs)

class Model(metaclass=ModelMeta):
    def save(self):
        print(f"保存到表: {self.table_name}")

class User(Model):
    def __init__(self, name):
        self.name = name

user = User("张三")
print(user.table_name)  # user
user.save()             # 保存到表: user
```

### 3. 数据验证

```python
class ValidatedMeta(type):
    def __new__(cls, name, bases, attrs):
        for key, value in attrs.items():
            if isinstance(value, property):
                # 自动添加验证逻辑
                pass
        return super().__new__(cls, name, bases, attrs)

class Person(metaclass=ValidatedMeta):
    def __init__(self, name, age):
        if not isinstance(name, str):
            raise TypeError("姓名必须是字符串")
        if not isinstance(age, int) or age < 0:
            raise ValueError("年龄必须是正整数")
        self.name = name
        self.age = age

person = Person("张三", 25)
# person = Person(123, 25)  # TypeError!
```

## 常见错误

### 1. 过度使用元类

```python
# 不好：简单问题复杂化
class SimpleMeta(type):
    pass

class Simple(metaclass=SimpleMeta):
    pass

# 好：使用装饰器或普通类
def simple_decorator(cls):
    return cls

@simple_decorator
class Simple:
    pass
```

### 2. 混淆 __new__ 和 __init__

```python
class MyClass:
    def __new__(cls):
        # 创建实例
        instance = super().__new__(cls)
        return instance

    def __init__(self):
        # 初始化实例
        self.value = 10
```

## 最佳实践

1. **谨慎使用元类**：99% 的情况不需要
2. **优先使用装饰器**：更简单直观
3. **使用类装饰器**：替代简单元类
4. **文档化用途**：说明为什么需要元类
5. **保持简单**：避免复杂的元类层次

```python
# 类装饰器替代元类
def add_method(cls):
    cls.new_method = lambda self: "New method"
    return cls

@add_method
class MyClass:
    pass

obj = MyClass()
print(obj.new_method())  # New method
```

# 闭包

[官方文档](https://docs.python.org/zh-cn/3/tutorial/classes.html#scopes-and-namespaces)

闭包是函数式编程中的重要概念，指内部函数引用了外部函数的变量。即使外部函数已经执行完毕，内部函数仍然可以访问这些变量。本文将详细介绍 Python 闭包的原理和应用。

## 闭包概念

### 什么是闭包？

闭包由三部分组成：
1. 内部函数
2. 外部函数的变量
3. 内部函数对外部变量的引用

```python
def outer(x):
    def inner(y):
        return x + y  # inner 引用了 outer 的变量 x
    return inner

closure = outer(10)
print(closure(5))  # 15
```

### 自由变量

在闭包中，被内部函数引用但未在内部定义的外部变量称为自由变量。

```python
def make_multiplier(factor):
    # factor 是自由变量
    def multiply(number):
        return number * factor
    return multiply

double = make_multiplier(2)
triple = make_multiplier(3)

print(double(5))  # 10
print(triple(5))  # 15
```

## 闭包的特性

### 1. 记住创建时的环境

```python
def create_greeter(greeting):
    def greeter(name):
        return f"{greeting}, {name}!"
    return greeter

hello_greeter = create_greeter("Hello")
hi_greeter = create_greeter("Hi")

print(hello_greeter("张三"))  # Hello, 张三!
print(hi_greeter("李四"))     # Hi, 李四!
```

### 2. 数据封装

```python
def create_counter():
    count = 0

    def increment():
        nonlocal count
        count += 1
        return count

    return increment

counter = create_counter()
print(counter())  # 1
print(counter())  # 2
print(counter())  # 3
```

## nonlocal 关键字

在闭包中修改外部变量需要使用 `nonlocal`。

```python
def outer():
    x = 10

    def inner():
        nonlocal x
        x = 20

    inner()
    print(x)  # 20

outer()
```

### 不使用 nonlocal 的情况

```python
def outer():
    x = 10

    def inner():
        # 这里创建了一个新的局部变量 x
        x = 20
        print(f"inner: {x}")

    inner()
    print(f"outer: {x}")

outer()
# inner: 20
# outer: 10
```

## 闭包应用场景

### 1. 装饰器

```python
def logger(func):
    def wrapper(*args, **kwargs):
        print(f"调用函数: {func.__name__}")
        result = func(*args, **kwargs)
        print(f"返回值: {result}")
        return result
    return wrapper

@logger
def add(a, b):
    return a + b

add(3, 5)
# 调用函数: add
# 返回值: 8
```

### 2. 回调函数

```python
def create_callback(message):
    def callback():
        print(message)
    return callback

on_success = create_callback("操作成功")
on_error = create_callback("操作失败")

# 模拟事件触发
on_success()  # 操作成功
on_error()    # 操作失败
```

### 3. 缓存

```python
def memoize(func):
    cache = {}

    def wrapper(*args):
        if args not in cache:
            cache[args] = func(*args)
        return cache[args]

    return wrapper

@memoize
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(50))  # 快速计算
```

### 4. 配置注入

```python
def create_api_client(base_url):
    def get(endpoint):
        url = f"{base_url}/{endpoint}"
        print(f"GET {url}")
        # 实际发送请求...
        return {"status": "ok"}

    def post(endpoint, data):
        url = f"{base_url}/{endpoint}"
        print(f"POST {url}")
        # 实际发送请求...
        return {"status": "created"}

    return {"get": get, "post": post}

client = create_api_client("https://api.example.com")
client.get("users")
client.post("users", {"name": "张三"})
```

## 闭包 vs 类

闭包和类都可以实现状态封装。

### 闭包实现

```python
def create_accumulator():
    total = 0

    def add(value):
        nonlocal total
        total += value
        return total

    return add

acc = create_accumulator()
print(acc(10))  # 10
print(acc(20))  # 30
```

### 类实现

```python
class Accumulator:
    def __init__(self):
        self.total = 0

    def add(self, value):
        self.total += value
        return self.total

acc = Accumulator()
print(acc.add(10))  # 10
print(acc.add(20))  # 30
```

> 💡 **选择建议**：简单状态用闭包，复杂逻辑用类。

## 查看闭包变量

```python
def outer(x):
    y = 10
    def inner(z):
        return x + y + z
    return inner

closure = outer(5)

# 查看自由变量
print(closure.__code__.co_freevars)  # ('x', 'y')

# 查看闭包单元
for cell in closure.__closure__:
    print(cell.cell_contents)
# 5
# 10
```

## 常见错误

### 1. 循环中的闭包陷阱

```python
# 错误示例
functions = []
for i in range(5):
    def func():
        return i
    functions.append(func)

print([f() for f in functions])  # [4, 4, 4, 4, 4]（都是 4）

# 正确做法：使用默认参数
functions = []
for i in range(5):
    def func(x=i):
        return x
    functions.append(func)

print([f() for f in functions])  # [0, 1, 2, 3, 4]
```

### 2. 可变对象的闭包

```python
def create_appenders():
    appenders = []
    lst = []
    for i in range(3):
        def appender():
            lst.append(i)
            return lst
        appenders.append(appender)
    return appenders

appenders = create_appenders()
print(appenders[0]())  # [0, 1, 2]
print(appenders[1]())  # [0, 1, 2, 1]
```

## 实际应用示例

### 1. 中间件模式

```python
def create_middleware(handler):
    def middleware(request):
        print("请求前处理")
        response = handler(request)
        print("响应后处理")
        return response
    return middleware

def handle_request(request):
    return f"处理请求: {request}"

wrapped = create_middleware(handle_request)
print(wrapped("GET /api/users"))
```

### 2. 权限检查

```python
def require_role(role):
    def decorator(func):
        def wrapper(user, *args, **kwargs):
            if user.get("role") != role:
                raise PermissionError(f"需要 {role} 权限")
            return func(user, *args, **kwargs)
        return wrapper
    return decorator

@require_role("admin")
def delete_user(admin, user_id):
    return f"删除用户 {user_id}"

admin = {"name": "管理员", "role": "admin"}
print(delete_user(admin, 123))
```

### 3. 重试机制

```python
def retry(max_attempts=3):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for attempt in range(1, max_attempts + 1):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    if attempt == max_attempts:
                        raise
                    print(f"尝试 {attempt} 失败: {e}")
        return wrapper
    return decorator

@retry(max_attempts=3)
def unstable_operation():
    import random
    if random.random() < 0.7:
        raise ConnectionError("连接失败")
    return "成功"
```

## 最佳实践

1. **谨慎使用 nonlocal**：优先考虑返回值
2. **注意循环闭包陷阱**：使用默认参数捕获变量
3. **保持闭包简单**：复杂逻辑考虑使用类
4. **文档化闭包用途**：说明自由变量的作用
5. **避免深层嵌套**：不超过 2-3 层
